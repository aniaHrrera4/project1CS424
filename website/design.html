According to the data set's ReadMe file :
"Ridership Readme
12-Aug-2011
Chicago Transit Authority

* About CTA ridership numbers *
Ridership statistics are provided on a system-wide and bus route/station-level basis. Ridership is primarily counted as boardings, that is, customers boarding a transit vehicle (bus or rail). On the rail system, there is a distinction between station entries and total rides, or boardings. Datasets indicate such in their file name and description.
* How people are counted on the 'L' *
On the rail system, a customer is counted as an "entry" each time he or she passes through a turnstile to enter a station. Customers are not counted as "entries" when they make a "cross-platform" transfer from one rail line to another, since they don't pass through a turnstile. Where the number given for rail is in "boardings," what's presented is a statistically valid estimate of the actual number of boardings onto the rail system.

* How people are counted on buses *
Boardings are recorded using the bus farebox and farecard reader. In the uncommon situation when there is an operating error with the farebox and the onboard systems cannot determine on which route a given trip's boardings should be allocated, these boardings are tallied as Route 0 in some reports. Route 1001 are shuttle buses used for construction or other unforeseen events.

* "Daytype" *
Daytype fields in the data are coded as "W" for Weekday, "A" for Saturday and "U" for Sunday/Holidays. Note that New Year's Day, Memorial Day, Independence Day, Labor Day, Thanksgiving, and Christmas Day are considered as "Sundays" for the purposes of ridership reporting. All other holidays are reported as the type of day they fall on."

Data Handling


Data was Downloaded as a CSV file,
then using the power of Python and Pandas the data was split into 3 specfic files based on station and read into R studio as data frames.

1. UIC-Halsted - uicData
2. O'Hare Airport - ohareData
3. Cermak-Chinatown - chinatownData

While the data was processed the original data was all turned to characters. So more fine tuning was done in R Studio a newDate, Month_,Year_,Rides2,year_month, and yearss, Columns were added. Rides2 the numeric version of rides and wis the column that will be used to vizualize all the data.

New data frames were created that aggregated the number of rides by month and year. (data_aggrUIC, data_aggrOhare, data_aggrChina) In the code above you can see how the data was handled and the resulting data frames.

After all the data handeling we are left with 3 main data frames from which the infomation was used to plot our bar charts.

Interesting Findings


Now that the application is live, we can use it to understand the data. The first thing I wanted to look at were the effects of the pandemic on each station. Image 1 shows the UIC-Halsted station for 2019, 2020, and 2021 2019 data shows the regular trends of rides spiking during the school months and dropping to a constistent rate during the summer monthso f May, June, July and the Holidays in December. In 2020 when the pandemic Hit you can clearly see that in march 2020 the rides perday dramatically dropped from 7000+ to less than 1000 from one day to the next. Then in the monthly chart you can see how the numbers stayed low during the toughest part of the pandemic when a vaccine was not yet produced from March 2020 - May 2021. In august 2021 classes were hybrid most still online however you can see how more and more riders were utilizing the UIC-Halsted station. These same trends can be seen in the O'Hare and Chinatown stations.
